## Release stage definitions

- **Live**: production-ready, stable, supported.
- **Beta**: available with known constraints.
- **Planned**: directional, not committed to production timelines.

Every roadmap claim in these docs should include a stage, expected impact, and dependency notes. Treat anything marked Planned as directional.

## Current focus

- Improve first-run onboarding and agent templates.
- Expand multi-agent operational controls.
- Strengthen trust layers: logging, provenance, and security controls.
- Grow the ecosystem of connectors and contributor pathways.

## Phases

### Alpha (current)

- Creator Agent, TAD, Orchestration, Execution Engine, ACP, MetaMind, DNAChain, and x402 available for early builders.
- Warren in early access for trading users.
- Marketplace browsing and listing flows live.

### Beta

- Public API and SDK reference.
- Expanded MCP tool catalog and bring-your-own MCP.
- Strategy marketplace moderation and creator royalties.

### GA

- SLA-backed execution for production workloads.
- Full multi-chain settlement through x402.
- Governance for listed agents and strategies.

## Before we publish a Live claim

1. Confirm capability in product.
2. Validate with one real workflow in a production-like environment.
3. Publish a changelog entry with date and scope.
